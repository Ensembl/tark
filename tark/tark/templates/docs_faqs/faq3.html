What additional services are provided by REST Tark?

- apply various filters and GET the results in JSON format when querying for existing transcripts and related entities
  1. Assembly
    1. Go to GET /api/assembly/
      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses based on assembly_name, enter GRCh37 or GRCh38 against the parameter and click Try it out! .
    2. Go to GET /api/assembly/{assembly_id}/
      1. In assembly_id field, enter 1 or 1001 to filter the response based on GRCh37 or GRCh38 respectively.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  2. Exon
    1. Go to GET /api/exon/
      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses, enter the filter value against the relevant parameter and click Try it out! .
        These parameters include page number, stable id (eg: ENSE00001184784), stable id version (eg: 1 or 5), start or end location, chromosomal region (eg: 1, 6, CHR_HSCHR7_2_CTG6, LRG_192), strand, assembly name (eg: GRCh37, GRCh38), release short name (eg: 90), source name (Ensembl or RefSeq).
      3. If you want to expand a specific object (sequence, session, assembly, exon_release_set or transcript) from the response body, enter that object in the expand field and click Try it out!.
    2. Go to GET /api/exon/{exon_id}/
      1. In exon_id field, enter an integer value, say 1.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  3. Gene
    1. Go to GET /api/gene/
      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses, enter the filter value against the relevant parameter and click Try it out! .
        These parameters include page number, stable id (eg: ENSG00000139618), stable id version (eg: 1 or 5), start or end location, chromosomal region (eg: 1, 6, CHR_HSCHR7_2_CTG6, LRG_192), strand, assembly name (eg: GRCh37, GRCh38), release short name (eg: 90), source name (Ensembl or RefSeq).
      3. If you want to expand a specific object (sequence, session, assembly, translation_release_set or transcripts) from the response body, enter that object in the expand field and click Try it out!.
    2. Go to GET /api/gene/{gene_id}/
      1. In gene_id field, enter an integer value, say 1.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  4. Release
    1. Go to GET /api/release/
      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses, enter the filter value against the relevant parameter and click Try it out! .
        These parameters include page number, assembly name (eg: GRCh37, GRCh38), release short name (eg: 90), source name (Ensembl or RefSeq).
      3. If you want to expand a specific object (eg: release_source or release_set) from the response body, enter that object in the expand field and click Try it out!.
    2. Go to GET /api/release/nopagination
      1. In page field, enter an integer value, say 1.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
    3. Go to GET /api/release/{release_id}/
      1. In release_id field, enter an integer value, say 1 or 1001.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  5. Transcript
    1. Go to GET /api/transcript/
      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses, enter the filter value against the relevant parameter and click Try it out! .
        These parameters include page number, stable id (eg: ENST00000252934), stable id version (eg: 1 or 5), start or end location, chromosomal region (eg: 1, 6, CHR_HSCHR7_2_CTG6, LRG_192), strand, assembly name (eg: GRCh37, GRCh38), release short name (eg: 90), source name (Ensembl or RefSeq).
      3. If you want to expand a specific object (sequence, session, assembly, transcript_release_set, genes, translations, exons) from the response body, enter that object in the expand field and click Try it out!.
    2. Go to GET /api/transcript/diff/
      1. In diff_me_stable_id field, enter a stable_id (eg:ENST00000380152).
      2. In diff_with_stable_id field, enter a stable_id (eg:ENST00000252934).
      3. If you want to further filter the responses, enter the filter value against the relevant parameter and click Try it out! .
        These parameters include page number, assembly name (eg: GRCh37, GRCh38), release short name (eg: 86, 88, 90), source name (Ensembl or RefSeq).
      4. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
    3. Go to GET /api/transcript/manelist/
      1. In page field, enter an integer value, say 1.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
    4. Go to GET /api/transcript/search/
      1. In identifier_field field, enter any valid identifier (eg:BRCA2, ENSG00000139618, ENST00000380152).
      2. In expand field, enter transcript_release_set or genes.
      3. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
    5. Go to GET /api/transcript/stable_id_with_version/
      1. In stable_id_with_version field, enter a stable id with its version, say ENST00000252934.10.
      2. In expand field, enter transcript_release_set or genes.
      3. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  6. Translation

      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses, enter the filter value against the relevant parameter and click Try it out! .
        These parameters include page number, stable id (eg: ENSP00000369497), stable id version (eg: 1 or 5), start or end location, chromosomal region (eg: 1, 6, CHR_HSCHR7_2_CTG6, LRG_192), strand, assembly name (eg: GRCh37, GRCh38), release short name (eg: 90), source name (Ensembl or RefSeq).
      3. If you want to expand a specific object (sequence, session, assembly, translation_release_set or transcripts) from the response body, enter that object in the expand field and click Try it out!.
    2. Go to GET /api/translation/{translation_id}/.
      1. In translation_id field, enter an integer value, say 1.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
