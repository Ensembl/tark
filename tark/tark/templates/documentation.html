TaRK FAQ

What is TaRK?

TaRK stands for Transcript ARchive. The goal of TArK is to create an archive of all transcript sequences from Ensembl and other sources, including historical gene sets. To ensure robust tracking of exact changes between gene and transcript sets, checksums are included at a fine grained level.

What services are provided by TaRK?

The TaRK web service would provide the following:
- querying of existing transcripts and related entities
- explore and determine the exact differences between transcripts from one release to the next

What attributes are captured for a given Transcript?

Attribute Type Mandatory Description Example
StableID String YES stable id of the transcript ENST00000387314
StableID Version Integer YES stable id version 2

What do you mean by Checksum? How is it derived?

A checksum is a small-sized datum derived from a block of digital data. The actual procedure which yields the checksum from a data input is called a checksum function or checksum algorithm, which usually output a significantly different value, even for small changes made to the input.

How to do GET operations via REST API?

GET
curl -X GET --header 'Accept: application/json' '{{hostname}}/api/transcript/'
curl -X GET --header 'Accept: application/json' '{{hostname}}/api/transcript/?stable_id=ENST00000639989'
curl -X GET --header 'Accept: application/json' '{{hostname}}/api/transcript/?stable_id=ENST00000639989&assembly_name=GRCh38'
curl -X GET --header 'Accept: application/json' '{{hostname}}/api/transcript/?stable_id=ENST00000639989&assembly_name=GRCh38&release_short_name=90'

What is the technology stack of TaRK?

Python3, Django, Django REST Framework and Django REST Swagger. Backend support is provided by MySQL.
