Tark FAQ

What is Tark?

Tark stands for Transcript ARchive. The goal of Tark is to create an archive of all transcript sequences from Ensembl and other sources, including historical gene sets. To ensure robust tracking of exact changes between gene and transcript sets, checksums are included at a fine grained level.

What services are provided by Tark?

Tark provides the following main services via REST Tark and Web Tark:

- search for existing transcripts and related entities
  1. Go to Tark Home
  2. In the Search field, enter a transcript (eg: NM_013236) and click Enter.
  3. The results page displays available versions of the selected transcript.
  4. In the Transcript StableID column, click on the version of your choice to view details of the transcript.
- explore and determine the exact differences between transcripts from one release to the next
  1. Go to Tark Home
  2. In Search field, enter a transcript (eg: NM_013236) and click Enter.
  3. The results page displays available versions of the transcript (eg: 3 versions of NM_013236).
  4. To compare any two versions of the transcript, say between NM_013236.4 and NM_013236.3:
    1. Click on T1 column against Transcript StableID NM_013236.3 (release: r106_20140203-Feb2014).
    2. Click on T2 column against Transcript StableID NM_013236.4 (release: r109_20190607-Jun2019).
    3. Click Compare Transcripts

What additional services are provided by REST Tark?

- apply various filters and GET the results in JSON format when querying for existing transcripts and related entities
  1. Assembly
    1. Go to GET /api/assembly/
      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses based on assembly_name, enter GRCh37 or GRCh38 against the parameter and click Try it out! .
    2. Go to GET /api/assembly/{assembly_id}/
      1. In assembly_id field, enter 1 or 1001 to filter the response based on GRCh37 or GRCh38 respectively.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  2. Exon
    1. Go to GET /api/exon/
      1. Click Try it out! under Response Messages, without filling any parameter to view the curl command, request url, response body, code and headers.
      2. If you want to filter the responses, enter the filter value against the relevant parameter and click Try it out! .
        These parameters include page number, stable id (eg: ENSE00001184784), stable id version (eg: 1 or 5), start or end location, chromosomal region (eg: 1, 6, CHR_HSCHR7_2_CTG6, LRG_192), strand, assembly name (eg: GRCh37, GRCh38), release short name (eg: 90), source name (Ensembl or RefSeq).
      3. If you want to expand a specific object (sequence, session, assembly, exon_release_set or transcript) from the response body, enter that object in the expand field and click Try it out!.
    2. Go to GET /api/exon/{exon_id}/
      1. In exon_id field, enter an integer value, say 1.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  3. Gene

      3. If you want to expand a specific object (sequence, session, assembly, translation_release_set or transcripts) from the response body, enter that object in the expand field and click Try it out!.
    2. Go to GET /api/gene/{gene_id}/
      1. In gene_id field, enter an integer value, say 1.
      2. Click Try it out! under Response Messages to view the curl command, request url, response body, code and headers.
  4. Release

What do you mean by Checksum? How is it derived?

A checksum is a small-sized datum derived from a block of digital data. The actual procedure which yields the checksum from a data input is called a checksum function or checksum algorithm, which usually output a significantly different value, even for small changes made to the input.

  public static void main(String[] args) throws Exception {

    String ext = "/api/assembly/";
    URL url = new URL(server + ext);

    URLConnection connection = url.openConnection();
    HttpURLConnection httpConnection = (HttpURLConnection)connection;
    
    httpConnection.setRequestProperty("Content-Type", "application/json");
    

    InputStream response = connection.getInputStream();
    int responseCode = httpConnection.getResponseCode();

    if(responseCode != 200) {
      throw new RuntimeException("Response code was not 200. Detected response was "+responseCode);
    }

    String output;
    Reader reader = null;
    try {
      reader = new BufferedReader(new InputStreamReader(response, "UTF-8"));
      StringBuilder builder = new StringBuilder();
      char[] buffer = new char[8192];
      int read;
      while ((read = reader.read(buffer, 0, buffer.length)) > 0) {
        builder.append(buffer, 0, read);
      }
      output = builder.toString();
    } 
    finally {
        if (reader != null) try {
          reader.close(); 
        } catch (IOException logOrIgnore) {
          logOrIgnore.printStackTrace();
        }
    }

    System.out.println(output);
  }
}
